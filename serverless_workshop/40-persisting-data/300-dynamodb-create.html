Infrastructure caveat

The general idea of this workshop is to show the basics of serverless app development with Python. Managing AWS infrastructure efficiently and with the correct tools is a secondary goal. That is why we will be using AWS CLI or simple Python scripts to provision infrastructure. Real-world projects would rely on some form of infrastructure-as-code (IaC), using tools like:

  • AWS CloudFormation
  • AWS CDK (Cloud Development Kit)
  • Terraform
  • Pulumi
  • Ansible

Create our DynamoDB table (AWS CLI)

aws dynamodb create-table --table-name $WORKSHOP_NAME-savealife-$ENV \
  --profile workshop \
  --attribute-definitions AttributeName=first_name,AttributeType=S \
  --key-schema AttributeName=first_name,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST

Create our DynamoDB table (Python SDK)

The following code is not needed if you already created the table using the AWS CLI. It serves to show how you could do it if you needed/wanted to.

Create ~/serverless_workshop/savealife/infra/create_dynamodb.py with the following contents:

import boto3

from os import getenv
from dotenv import load_dotenv

load_dotenv()

ENV = getenv("ENV", "dev")
client = boto3.client("dynamodb")
first_name = getenv("WORKSHOP_NAME", "ivica")  # replace with your own name of course
table_name = f"{first_name}-savealife-{ENV}"

try:
    client.create_table(
        TableName=table_name,
        AttributeDefinitions=[{"AttributeName": "first_name","AttributeType": "S"}],
        KeySchema=[{"AttributeName": "first_name", "KeyType": "HASH"}],
        BillingMode="PAY_PER_REQUEST",
        Tags=[
            {
                "Key": "env",
                "Value": ENV,
            },
            {
                "Key": "project",
                "Value": "serverless_workshop",
            },
            {
                "Key": "owner",
                "Value": first_name,
            },
        ]
    )
    print(f"Table {table_name} created successfully!")

except Exception as err:
    print(f"Error creating table: {err}")

With this piece of code we will create a DynamoDB table with:

  • The primary key being first_name. We talked about primary keys on the DynamoDB 101 page.
  • The BILLING_MODE set to PAY_PER_REQUEST. This basically means that our table will scale to $0 if there is no usage. It can also scale to $MANY if there is a lot of usage. Know your data access patterns.

Create the table

Run python infra/create_dynamodb.py which should result in

python infra/create_dynamodb.py

will result in something similar to:

Table ivica-savealife-dev created successfully!

This step is crucial, please don’t proceed further if this step failed and ask for assistance.
