Project setup

For this workshop we will be using AWS Chalice - a framework for writing serverless, API first and/or event driven applications. Anyone familiar with Flask or FastAPI should feel right at home with the decorator-based API of Chalice.

I chose it because it is simple enough to get started in minutes and yet powerful enough to give you a taste of writing scalable apps on AWS. Developers like using it because it takes care of the heavy lifting for you and provisions (most of) the infrastructure needed.

It provides a very good developer experience since everything, the application and it’s infrastructure can be deployed right from your machine, or from a CI/CD system, with a single command.

Install Chalice

Python 3.6+ recommended, up to and including 3.9.

{
    mkdir ~/serverless_workshop && cd ~/serverless_workshop
    python -m pip install virtualenv
    python -m virtualenv venv
    source venv/bin/activate
    python -m pip install chalice==1.27.0
}

MacOS users might need to create a symlink to the python3 executable: ln -s /usr/local/bin/python3 /usr/local/bin/python

The end result should be similar to this:

chalice --version

Outputs:

chalice 1.27.0, python 3.10.2, linux 5.16.15-arch1-1

Create a project

Our Chalice project will be called savealife.

chalice new-project savealife

If you get a warning about using a newer Python version than supported on AWS Lambda please ignore it.

Explore the savealife folder

Looking at the savealife folder we can observe the basic Chalice skeleton:

├── app.py
├── .chalice
    └── config.json
├── .gitignore
└── requirements.txt

As you can probably guess yourself, the dependencies of your app should be listed in requirements.txt, your Python code goes to app.py and .chalice/config.json is used for various configuration options supported by Chalice.

Install requirements

Assuming you’re in ~/serverless_workshop/savealife/, this is how you can create the basic requirements file, after which you can install it the usual way:

cat > requirements.txt <<EOF
boto3==1.22.2
botocore==1.25.2
python-dotenv==0.20.0
EOF

Configure .env

To make use of environment variables needed for development we will be using the python-dotenv project. In short, it allows us to define arbitrary environment variables through the .env file, like so:

cat > .env <<EOF
WORKSHOP_NAME="YOUR_FIRST_NAME_HERE"
ENV=dev
AWS_PROFILE=workshop
AWS_DEFAULT_REGION=eu-central-1
EOF

Please create this file in the root of your project (~/serverless_workshop/savealife/.env) and set the variables as shown above, replacing your first name of course.

It is very important that you make use of these variables, like so:

{
    set -a
    source .env
    set +a
}

Verify that environment variables are set correctly:

env | grep -ie '\(^WORK\|^ENV\|^AWS\)'

WORKSHOP_NAME=ivica
AWS_PROFILE=workshop
AWS_DEFAULT_REGION=eu-central-1
ENV=dev

This step is needed in every terminal window/application you may use in this workshop. Not having the environment variables present will make certain commands/actions to fail.

Configure Chalice with your first name

Many of us will be working in the same AWS account and we need a way to distinguish who created what. To make that easy, edit ~/serverless_workshop/savealife/.chalice/config.json to contain your first name, like so:

cat > ./savealife/.chalice/config.json <<EOF 
{
  "version": "2.0",
  "app_name": "YOUR_FIRST_NAME_HERE-savealife",
  "stages": {
    "dev": {
      "api_gateway_stage": "api"
    }
  }
}
EOF
